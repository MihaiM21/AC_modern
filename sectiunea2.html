II. FORMATUL INSTRUCȚIUNILOR

Formatul instrucțiunilor se reprezintă de obicei în formă grafică sub forma unor dreptunghiuri ce simbolizează câmpurile binare incluse în instrucțiuni. Din punctul de vedere al lungimii instrucțiunilor pot exista două tipuri de ASI:

Lungime Fixă

Arhitecturi la care lungimea instrucțiunilor este fixă, pentru tot setul recunoscut.

Exemple: arhitecturile RISC, dar și arhitecturi CISC cum ar fi cea a calculatorului Felix C256.

Lungime Variabilă

Arhitecturi cu lungime variabilă a instrucțiunilor recunoscute.

La microprocesoarele cu lungime fixă, unitatea de control este mai simplă, dar densitatea codului este mai mică.

Tipuri de Arhitecturi

În funcție de locul de stocare a operanzilor, există trei tipuri principale de arhitecturi:

Tip Denumire Caracteristici Exemplu Instrucțiune
Ac Arhitectură pe bază de acumulator Acumulatorul este referit ca registru implicit add x
Ac ← Ac + x
RG Arhitectură pe bază de registre de uz general Operanzi în registre, ușor de generat cod add r1,r2,r3
r1 ← r2 + r3
ST Arhitectură pe bază de stivă Instrucțiuni cu zero adrese, notație poloneză push x
add
VS ← @x
VS ← (b + c)

Avantaje și Dezavantaje

Comparație Arhitecturi

Arhitectură pe bază de acumulator (Ac)
  • Avantaje: Instrucțiuni scurte, densitate relativ mare a codului
  • Dezavantaje: Acumulatorul constituie un puternic factor de limitare al vitezei de lucru
Arhitectură pe bază de registre de uz general (RG)
  • Avantaje: Ușurință generării codului de către programe compilatoare, viteza de prelucrare ridicată
  • Dezavantaje: Lungimea instrucțiunilor este relativ mare (fiecare registru trebuie adresat)
Arhitectură pe bază de stivă (ST)
  • Avantaje: Evaluare extrem de ușoară a operațiilor pe baza notației inverse (notația „poloneză"), instrucțiuni cu lungime foarte redusă
  • Dezavantaje: Stiva este mai dificil de adresat în mod aleator, codul rezultat este relativ ineficient, limitare în viteză

Exemplu: Evaluarea Expresiei x = a × (b + c)

Pentru a ilustra influența numărului de adrese în programele executate de procesor vom evalua egalitatea:

x = a × (b + c)

Presupunem că operanzii a, b și c din relație sunt stocați în memorie, în locațiile de adrese a, b, c, iar rezultatul se va introduce la adresa x în memorie.

a) Arhitecturi pe baza de registre de uz general

Instrucțiuni cu 3 adrese (CISC)
add r1, b, c ; r1 ← @b + @c
mul x, r1, a ; @x ← r1 * @a

Observație: Deși programul este foarte scurt, el conține instrucțiuni cu format de lungime mare și se desfășoară relativ lent. La prima instrucțiune se fac două accesări la memorie pentru citirea operanzilor b și c. La fel pentru a doua instrucțiune sunt necesare două accesări succesive la memorie (operand și rezultat).

Instrucțiuni cu 3 adrese (RISC)
load r1, a ; r1 ← @a
load r2, b ; r2 ← @b
load r3, c ; r3 ← @c
add r2, r2, r3 ; r2 ← (r2 + r3)
mul r1, r1, r2 ; r1 ← (r1 * r2)
store x, r1 ; @x ← r1

Avantaj: La acest tip de arhitecturi se folosesc doar instrucțiuni de load (încarcă) și store (stochează) când se comunică cu memoria, iar toate celelalte instrucțiuni se referă la operanzi din registrele UCP (deci prelucrare foarte rapidă).

Instrucțiuni cu 2 adrese (CISC)
load r1, b ; r1 ← @b
add r1, c ; r1 ← r1 + @c
mul r1, a ; r1 ← r1 * @a
store x, r1 ; @x ← r1

b) Arhitecturi pe baza de acumulator

Instrucțiuni cu o adresă, acumulatorul fiind adresat implicit:

load b ; ac ← @b
add c ; ac ← ac + @c
store t ; @t ← ac
load a ; ac ← @a
mul t ; ac ← ac * @t
store x ; @x ← ac

unde s-a notat cu „t" adresa unei locații de memorie folosită pentru stocarea temporară a rezultatelor intermediare

c) Arhitecturi pe bază de stivă

Instrucțiuni cu zero adrese:

push a ; vs ← @a, la adr
push b ; vs ← @b, la adr-1
push c ; vs ← @c, la adr-2
add ; vs ← (b + c), stocat la adr-1
mul ; vs ← a*(b+c), stocat la adr
pop x ; @x ← vs, vs pointează la adr+1

Notație: Pentru a evalua expresiile aritmetice într-un calculator stivă, e necesar să se convertească expresia în notația poloneză (inversă). Ideea a pornit de la matematicianul polonez Lukasievics J, care a notat o expresie de forma "a + b" ca "ab+".

Notația Poloneză

După același principiu putem exemplifica notația poloneză și pentru alte ecuații:

Expresie Notație Poloneză
x = a × (b + c) abc+×
x = (a + b) × (c + d) ab + cd+×
x = (a + b) × (c + d) × (e + f) ab + cd + ef+××

Avantaj: Notația poloneză are avantajul că nu folosește paranteze. Ba mai mult, pornind de la stânga la dreapta se indică toate operațiile ce trebuie efectuate, în ordine, pe o arhitectură de tip stivă.
